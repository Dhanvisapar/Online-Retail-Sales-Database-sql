# Online-Retail-Sales-Database-sql

## Introduction
This project focuses on designing a relational database for an online retail platform to manage customers, products, orders, and sales data.

## Abstract
The database is normalized to reduce redundancy and supports analytical queries to generate sales insights such as total customer spending and top-selling products.

## Tools Used
- MySQL
- MySQL Workbench / DBeaver

## Database Structure
- Customers
- Products
- Orders
- Order_Items

## Key Features
- Normalized database design (3NF)
- Sample e-commerce sales data
- JOIN-based analytical queries
- Sales and product performance reports

## Steps Involved
1. Identified core e-commerce entities
2. Designed relational schema with foreign keys
3. Inserted sample customer and order data
4. Wrote JOIN queries for sales analysis
5. Generated summary reports

## Conclusion
This project demonstrates core SQL concepts such as joins, aggregation, and schema design, making it suitable for understanding real-world retail database systems.
